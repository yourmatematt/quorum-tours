# Build Prompt — Quorum Tours — TOUR-NOT-INCLUDED-01

## Context

The tour page currently shows an "Included" section listing what's covered in the tour price. There is no corresponding "Not Included" section. This creates ambiguity — guests may assume accommodation and travel are bundled.

## Task

Wherever "What's Included" (or equivalent) is displayed on the tour page, add a "Not Included" section immediately below it.

## Not Included content

Render the following as a list:

- Accommodation
- Travel to Mallacoota
- Travel insurance
- Meals not specified in the itinerary

## Data source

Add a `not_included` JSONB column to the `tours` table (if it doesn't already exist) following the same pattern as the `included` column.

Seed the Wilderness Coast Weekender tour (`slug: wilderness-coast-weekender-apr-2026`) with:

```json
[
  "Accommodation",
  "Travel to Mallacoota",
  "Travel insurance",
  "Meals not specified in the itinerary"
]
```

If adding a database column is out of scope for this task, hardcode the four items above as a fallback and note it for follow-up.

## Visual treatment

- Section heading: "Not Included"
- Use a subtle visual distinction from the Included section — muted text or a neutral icon (e.g. X or dash) rather than the green checkmarks used for inclusions
- Same card/container as the Included section, either directly below it or as a second column on wider screens
- Do not make it alarming — this is standard tour transparency, not a warning

## Locations to update

- Tour detail page (`/tours/[slug]`)
- Booking confirmation email (if inclusions are listed there)
- Any tour summary cards that list inclusions

## Validation
