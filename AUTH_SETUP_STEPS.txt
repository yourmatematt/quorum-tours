SUPABASE AUTHENTICATION - NEXT STEPS
====================================

You have a complete, production-ready Supabase authentication system.
Follow these steps to activate it.

STEP 1: GET YOUR SUPABASE CREDENTIALS (1 minute)
================================================

1. Go to https://app.supabase.com
2. Select "hello@quorumtours.com's Project"
3. Click "Settings" in the left sidebar
4. Click "API" in the settings menu
5. Copy these two values:
   - Project URL (looks like: https://xvfevfvqqlonzoxbehrc.supabase.co)
   - Anon public key (long string starting with eyJ...)

STEP 2: SET ENVIRONMENT VARIABLES (1 minute)
==============================================

Option A: Command line
  1. In terminal, run: cp .env.example .env.local
  2. Open .env.local in your editor
  3. Paste your Project URL after NEXT_PUBLIC_SUPABASE_URL=
  4. Paste your Anon key after NEXT_PUBLIC_SUPABASE_ANON_KEY=
  5. Save the file

Option B: Manual
  1. Copy .env.example to .env.local
  2. Edit .env.local
  3. Replace placeholder values with your credentials

STEP 3: RESTART DEV SERVER (30 seconds)
========================================

1. Stop your dev server (Ctrl+C)
2. Run: npm run dev
3. Wait for "Ready in X.XXs"

STEP 4: TEST AUTHENTICATION (2 minutes)
========================================

1. Open http://localhost:3000/signup
2. Enter an email: test@example.com
3. Enter password: TestPassword123!
4. Click "Create account"
5. You should see a success message

6. Go to http://localhost:3000/login
7. Log in with the same credentials
8. You should be logged in and at the home page

9. Refresh the page - you should stay logged in

10. Look for a logout button or link
11. Click logout
12. Try to access /profile - you should be redirected to login

DONE! Basic authentication is working.

OPTIONAL: SET UP EMAIL CONFIRMATION
====================================

Want users to confirm their email before login?

1. Go to https://app.supabase.com
2. Select your project
3. Go to "Authentication" > "Email Templates"
4. Click on "Confirm signup" template
5. Find this line:
   {{ .ConfirmationURL }}
6. Replace it with:
   {{ .SiteURL }}/auth/callback?token_hash={{ .TokenHash }}&type=email_change
7. Click "Save"
8. Test by creating a new account - you'll receive a confirmation email

OPTIONAL: SET UP GOOGLE OAUTH
==============================

Want "Sign in with Google" button?

1. Create OAuth credentials in Google Cloud Console:
   https://console.cloud.google.com
   - Create new project
   - Enable Google+ API
   - Create OAuth 2.0 credentials (Web application)
   - Add authorized redirect URI:
     https://xvfevfvqqlonzoxbehrc.supabase.co/auth/v1/callback
     (Replace the ID with your Supabase project ID)

2. In Supabase Dashboard:
   - Go to Authentication > Providers
   - Click Google
   - Enable it
   - Paste Client ID and Client Secret
   - Save

3. Test: Click "Sign in with Google" on login page

DEPLOYMENT
==========

When deploying to production:
1. Add same environment variables to your host (Vercel, etc.)
2. Update OAuth redirect URIs to production domain
3. Test all auth flows in production
