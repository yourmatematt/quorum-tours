**Document Type:** Information Architecture Specification
**Component:** Operator Dashboard - Profile View
**Status:** Draft
**Created:** 2026-01-22
**Design System:** Organic Biophilic (HOME-REDESIGN-DECISIONS.md)
**Target Users:** Tour operators aged 50-70
**Complexity:** Medium-High
**Estimated Build:** 4-5 days

---

## Purpose

The Profile view allows tour operators to edit their public operator profile, manage account settings, and configure business information. This addresses operator pain points around maintaining accurate, trustworthy profiles that build credibility with potential participants.

**Key User Goals:**
- Update public profile information (bio, specialties, credentials)
- Upload and manage profile photo
- Preview how profile appears to users
- Manage account security (password, email)
- Configure notification preferences
- Update business details (company, location, insurance)

**Element:** Photo preview with upload/remove buttons

**Behavior:**
- Click "Upload New Photo" â†’ File picker (accepts .jpg, .png, .webp)

**Default Avatar:**
- Initials-based (e.g., "RC" for Roberto Clay)
- Background: `bg-[var(--color-primary)]`

- File size < 5MB

#### Basic Information Fields

**Full Name (Required):**
- Type: Text input
- Validation: 2-100 characters, no special characters
- Error: "Full name is required"
- This name appears on booking confirmations and legal documents

**Display Name (Optional):**
- Type: Text input
- Validation: 2-50 characters
- Help text: "This name appears on your public profile"
- If empty, uses Full Name

**Headline:**
- Type: Text input
- Max length: 100 characters
- Character counter shown
- Help text: "A brief description of your expertise"
- Example: "Expert Birding Guide Specializing in Andean Ecosystems"

#### About Me

**Element:** Rich text editor with basic formatting

**Features:**
- Bold, italic, underline
- Bullet lists, numbered lists
- Links (auto-detected)
- Line breaks
- NO images, NO headings (to maintain consistent styling)

**Max Length:** 2000 characters (with character counter)

**Help Text:**
- "Tell participants about your background, experience, and guiding philosophy."
- "This appears on your public operator profile."

**Accessibility:**
- WYSIWYG editor with keyboard shortcuts
- Toolbar buttons have clear labels
- Screen reader announces formatting changes

#### Expertise & Specialties

**Primary Regions:**
- Type: Tag input with autocomplete
- Data source: Predefined list of regions
- Max: 5 regions
- Allows custom regions
- Examples: "Andes", "Patagonia", "Atlantic Forest", "Pantanal"

**Target Species Expertise:**
- Type: Tag input with eBird species autocomplete
- Search: eBird API for species names
- Max: 10 species
- Shows scientific name on hover
- Examples: "Andean Condor", "Torrent Duck", "Royal Cinclodes"

**Languages Spoken:**
- Type: Checkboxes
- Options: English, Spanish, Portuguese, French, German, Other
- "Other" includes text input for additional languages

**Years of Experience:**
- Type: Number input
- Min: 0, Max: 99
- Suffix: "years"
- Help text: "Total years guiding birding tours"

### Component Details

#### Operating As

**Radio Buttons:**
- Individual Guide (sole proprietor)
- Registered Company (LLC, corporation, etc.)

**Conditional Fields:**
- If "Registered Company": Show Company Name + Tax ID fields
- If "Individual Guide": Hide company fields

---

#### Company Details

**Company Name:**
- Required if "Registered Company"
- Appears on booking invoices
- Help text explains usage

**Tax ID / Business Registration:**
- Private field (not shown on public profile)
- Format varies by country
- Examples: CUIT (Argentina), EIN (USA), VAT (Europe)
- Optional but recommended for tax reporting

---

#### Base of Operations

**Primary Location:**
- Public field (shown on operator profile)
- City, Country format
- Helps participants understand operator's local expertise

**Mailing Address:**
- Private (used for payments, legal)
- Full address fields:
  - Street address
  - City
  - Province/State
  - Postal Code
  - Country

---

#### Insurance & Safety

**Liability Insurance Checkbox:**
- If checked: Show insurance details fields
- If unchecked: Hide details, show encouragement message

**Insurance Fields (all optional):**
- Provider name
- Policy number
- Coverage amount (currency + amount)
- Valid until date
- Certificate upload (PDF, 2MB max)

**Privacy:**
- All insurance info is private
- Used by Quorum Tours for operator verification
- May be shown to participants as "Insured Operator" badge (without details)

---

#### Emergency Contact

**Purpose:** In case operator has emergency during tour

**Fields:**
- Name (required)
- Relationship (optional)
- Phone (required)
- Email (optional)

**Privacy:** All private, only accessed in emergencies
